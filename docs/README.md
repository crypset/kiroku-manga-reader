# Kiroku Manga Reader

Kiroku Manga Reader — простий Android-застосунок для читання манґи, що
зберігається локально на пристрої. Застосунок не завантажує контент із мережі:
користувач обирає кореневу директорію, після чого застосунок показує знайдені
манґи, глави та сторінки.

## Можливості

- вибір директорії через Android Storage Access Framework;
- збереження дозволу на читання вибраної директорії;
- SQLite-кеш бібліотеки після повного сканування;
- відновлення списку манґи та глав після перезапуску без повторного сканування;
- ручний rescan поточної бібліотеки;
- очищення кешу бібліотеки без очищення прогресу читання;
- підтримка манґи з окремими директоріями глав і без них;
- природне сортування назв (`chapter 2` перед `chapter 10`);
- читання сторінок горизонтальними свайпами;
- масштабування pinch-to-zoom і подвійним дотиком;
- блокування свайпу між сторінками, коли поточна сторінка збільшена;
- приховування панелі читача дотиком;
- автоматичне збереження прогресу читання;
- налаштування ширини контейнера сторінки від `50%` до `200%`;
- асинхронне й паралельне сканування файлів.

## Вимоги до структури бібліотеки

Застосунок підтримує два варіанти організації файлів.

### Манґа з главами

```text
Manga Library/
├── Manga A/
│   ├── Chapter 1/
│   │   ├── 001.jpg
│   │   └── 002.jpg
│   └── Chapter 2/
│       ├── 001.webp
│       └── 002.webp
└── Manga B/
    └── Chapter 1/
        └── page.png
```

### Манґа без окремих глав

Якщо в директорії манґи немає вкладених директорій, усі зображення в ній
вважаються однією главою.

```text
Manga Library/
└── One Shot/
    ├── 001.jpg
    ├── 002.jpg
    └── 003.jpg
```

Підтримувані розширення: `png`, `jpg`, `jpeg`, `webp`, `gif`.

## Використання

1. Запустіть застосунок.
2. Натисніть плаваючу кнопку вибору директорії.
3. Надайте застосунку доступ до файлів, якщо Android його запитає.
4. Виберіть кореневу директорію, яка містить директорії манґи.
5. Виберіть манґу, потім главу.
6. Гортайте сторінки свайпами. Використовуйте pinch-to-zoom або подвійний дотик
   для масштабування.

Після сканування список манґи, глави та сторінки зберігаються локально в SQLite.
Під час наступного запуску застосунок відновлює бібліотеку з кешу. Щоб вибрати
іншу бібліотеку або оновити список після зміни файлів, відкрийте меню бібліотеки
та виберіть зміну папки або rescan.

Під час масштабування свайпи між сторінками блокуються, щоб жест рухав поточне
зображення. Одинарний дотик по сторінці показує або приховує панель читача.
Прогрес читання зберігається автоматично.

## Основні модулі коду

- `MainActivity`, `ChaptersActivity`, `ReaderActivity` — три основні екрани.
- `LibraryScanner` — сканування SAF/`DocumentFile`.
- `LibraryStore` і `LibraryDatabaseSchema` — SQLite-кеш бібліотеки.
- `ReadingProgressStore` — прогрес читання.
- `ReaderSettingsStore` — налаштування ширини сторінки.
- `MangaReaderIntents` — navigation extras між екранами.
- `NaturalSort` і `ImageFileType` — чисті правила для файлової логіки.

## Поточні обмеження

- Зміни у файловій системі не з'являться до ручного rescan бібліотеки.
- Підтримуються лише директорії та окремі файли зображень; архіви `CBZ`, `ZIP`
  та `RAR` не підтримуються.
- Список URI сторінок передається в читач через `Intent`; дуже великі глави
  можуть наблизитися до ліміту Binder transaction.
- Немає закладок, історії читання й пошуку по бібліотеці.
- Частина UI-рядків в адаптерах усе ще задана в коді та потребує винесення в
  `strings.xml`.
- Instrumented/UI-тести поки відсутні.
