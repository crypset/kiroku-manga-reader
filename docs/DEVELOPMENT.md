# Розробка

## Технології

| Компонент | Використання |
| --- | --- |
| Kotlin | прикладна логіка |
| Android Views та XML | інтерфейс користувача |
| RecyclerView | списки манґи та глав |
| ViewPager2 | гортання сторінок |
| DocumentFile / SAF | доступ до вибраних директорій |
| SQLiteOpenHelper | локальний кеш бібліотеки |
| SharedPreferences | прогрес читання та налаштування читача |
| Kotlin Coroutines | фонове й паралельне сканування файлів |
| Glide | завантаження зображень |
| PhotoView | масштабування сторінок |
| JUnit 4 | unit-тести чистої логіки |

Основні параметри Android:

- `minSdk`: 24;
- `targetSdk`: 34;
- `compileSdk`: 34;
- JVM target: Java 8;
- модуль застосунку: `app`;
- application ID: `com.crypset.kiroku.mangareader`.

## Підготовка середовища

Потрібні:

- Android Studio з підтримкою Kotlin;
- Android SDK 34;
- JDK, сумісний з Android Gradle Plugin, указаним у
  `gradle/libs.versions.toml`;
- Android-пристрій або емулятор з API 24 чи новішим.

Відкрийте корінь репозиторію в Android Studio та дочекайтеся Gradle Sync.
Локальний шлях до SDK зберігається в `local.properties`; цей файл не потрібно
комітити.

На Windows:

Debug APK після збірки розташовується в

## Відповідальності файлів

### Екрани

- `MainActivity` керує UI бібліотеки, picker-директорією, меню бібліотеки та
  запуском повного сканування.
- `ChaptersActivity` керує UI глав і вирішує, взяти глави з кешу чи просканувати
  вибрану манґу.
- `ReaderActivity` керує читанням сторінок, панеллю, прогресом і налаштуванням
  ширини контейнера.

### Сканування та правила файлів

- `LibraryScanner` — єдине місце, де слід змінювати логіку обходу
  `DocumentFile`.
- `NaturalSort` — природне сортування назв. Нові правила сортування додавайте
  тут, а не в Activity.
- `ImageFileType` — підтримувані розширення зображень. Додаючи формат,
  перевірте, що його підтримує Glide.

### Зберігання

- `LibraryStore` читає й записує доменні моделі в SQLite.
- `LibraryDatabaseSchema` містить таблиці, колонки та DDL.
- `ReadingProgressStore` зберігає прогрес читання.
- `ReaderSettingsStore` зберігає налаштування ширини сторінки.

### Контракти й моделі

- `MangaReaderIntents` — єдине місце для navigation extras.
- `MangaModels` містить `MangaItem` і `Chapter`.
- `ReadingProgressModels` містить `ChapterReadingProgress` і
  `MangaReadingProgress`.

## Як працювати з файловою бібліотекою

Для доступу до файлів застосунок використовує `OpenDocumentTree` і
`DocumentFile`, тому сканування має працювати з URI, а не з абсолютними шляхами
файлової системи.

Після вибору директорії `MainActivity` викликає
`takePersistableUriPermission()`. Будь-які нові функції, що працюють із
контентом, мають зберігати цей підхід і читати файли через `ContentResolver` або
API, сумісні зі Storage Access Framework.

Повне сканування виконується через `LibraryScanner.scanLibrary()`:

1. коренева директорія перетворюється на список директорій манґи;
2. кожна манґа сканується на глави;
3. якщо директорій глав немає, зображення в директорії манґи вважаються однією
   главою;
4. сторінки фільтруються через `ImageFileType`;
5. усі рівні сортуються через `NaturalSort`.

Результат зберігається в SQLite через `LibraryStore.saveLibrary()`. Повторний
вибір директорії або дія `Rescan library` запускають нове сканування та
перезаписують кеш бібліотеки.

## Сортування та підтримувані файли

Манґи, глави та сторінки сортуються природно: числові частини назви
порівнюються як числа. Наприклад:

```text
1.jpg
2.jpg
10.jpg
```

Підтримувані розширення зараз задаються в `ImageFileType`: `png`, `jpg`,
`jpeg`, `webp`, `gif`.

## Локальні дані

### Бібліотека

Бібліотека кешується в SQLite. Схема описана в `LibraryDatabaseSchema`, а доступ
до даних — у `LibraryStore`.

При зміні схеми потрібно:

1. збільшити `DATABASE_VERSION`;
2. додати міграцію в `onUpgrade()` або явно погодитися на втрату кешу;
3. перевірити сценарії першого запуску, rescan і clear cache.

### Прогрес читання

Прогрес зберігається автоматично при виборі сторінки в `ReaderActivity`.
Очищення прогресу доступне з меню бібліотеки та з діалогу налаштувань читача.
Очищення кешу бібліотеки не видаляє прогрес.

### Налаштування читача

Ширина контейнера сторінки зберігається через `ReaderSettingsStore`.
Діапазон: `50%..200%`, значення за замовчуванням: `100%`.

## Перевірка змін

Автоматична перевірка:

```powershell
.\gradlew.bat assembleDebug
.\gradlew.bat testDebugUnitTest
```

Мінімальний ручний сценарій:

1. Зібрати й запустити застосунок на API 24-29 та API 30+.
2. Вибрати бібліотеку з кількома манґами.
3. Перезапустити застосунок і перевірити, що список манґи відновився без
   повторного вибору директорії.
4. Відкрити манґу після перезапуску й перевірити, що глави доступні з кешу.
5. Запустити `Rescan library` і перевірити оновлення кешу.
6. Перевірити природне сортування назв із числами.
7. Відкрити манґу з директоріями глав.
8. Відкрити манґу без директорій глав.
9. Перевірити свайпи, pinch-to-zoom, подвійний дотик і приховування панелі.
10. Перевірити зміну ширини контейнера `50%`, `100%`, `150%`, `200%`.
11. Перезапустити застосунок і перевірити, що прогрес читання та ширина
    контейнера збереглися.
12. Перевірити порожні директорії та директорії з непідтримуваними файлами.

## Тести

Активні unit-тести лежать у
`app/src/test/java/com/crypset/kiroku/mangareader/`.

Поточне покриття:

- `NaturalSortTest` перевіряє природне сортування числових частин назв.
- `NaturalSortTest` перевіряє підтримувані та непідтримувані розширення
  зображень.

Найкорисніші наступні тести:

- сканування one-shot манґи без директорій глав;
- сканування манґи з кількома директоріями глав;
- збереження й відновлення `LibraryStore`;
- агрегація `MangaReadingProgress`.

## Відомі технічні особливості

- Activity-класи самостійно керують `CoroutineScope`; scope скасовується в
  `onDestroy`.
- `ReaderActivity` використовує deprecated fullscreen flags
  (`systemUiVisibility`). Це працює, але в майбутньому варто перейти на сучасні
  `WindowInsetsController` API.
- Список URI сторінок передається в `ReaderActivity` через `Intent`; для дуже
  великих глав це може наблизитися до ліміту Binder transaction.
- Застосунок працює з файлами через SAF і не має покладатися на абсолютні шляхи
  або повний доступ до всіх файлів. Перед публікацією в Google Play політику
  дозволів усе одно слід переглянути.
- Частина старих коментарів у Kotlin/XML може мати пошкоджене кодування; це не
  впливає на виконання, але ускладнює підтримку.
