# Розробка

## Технології

| Компонент | Використання |
| --- | --- |
| Kotlin | прикладна логіка |
| Android Views та XML | інтерфейс користувача |
| RecyclerView | списки манґи та глав |
| ViewPager2 | гортання сторінок |
| DocumentFile / SAF | доступ до вибраних директорій |
| SharedPreferences | локальний кеш URI бібліотеки та списку манґи |
| Kotlin Coroutines | фонове сканування файлів |
| Glide | завантаження зображень |
| PhotoView | масштабування сторінок |

Основні параметри Android:

- `minSdk`: 24;
- `targetSdk`: 34;
- `compileSdk`: 34;
- JVM target: Java 8;
- модуль застосунку: `app`;
- application ID: `com.crypset.kiroku.mangareader`.

## Підготовка середовища

Потрібні:

- Android Studio з підтримкою Kotlin;
- Android SDK 34;
- JDK, сумісний з Android Gradle Plugin, указаним у
  `gradle/libs.versions.toml`;
- Android-пристрій або емулятор з API 24 чи новішим.

Відкрийте корінь репозиторію в Android Studio та дочекайтеся Gradle Sync.
Локальний шлях до SDK зберігається в `local.properties`; цей файл не потрібно
комітити.

## Команди

На Windows:

```powershell
.\gradlew.bat assembleDebug
.\gradlew.bat installDebug
.\gradlew.bat lint
.\gradlew.bat test
```

На Linux або macOS:

```bash
./gradlew assembleDebug
./gradlew installDebug
./gradlew lint
./gradlew test
```

Debug APK після збірки розташовується в
`app/build/outputs/apk/debug/app-debug.apk`.

## Як працювати з файловою бібліотекою

Для доступу до файлів застосунок використовує `OpenDocumentTree` і
`DocumentFile`, тому сканування має працювати з URI, а не з абсолютними шляхами
файлової системи.

Після вибору директорії `MainActivity` викликає
`takePersistableUriPermission()`. Будь-які нові функції, що працюють із
контентом, мають зберігати цей підхід і читати файли через `ContentResolver` або
API, сумісні зі Storage Access Framework.

Після успішного сканування `MainActivity` зберігає URI кореневої директорії та
список `MangaItem` у `SharedPreferences`. На наступному запуску список
відновлюється з кешу без доступу до файлової системи. Повторний вибір директорії
кнопкою `+` запускає нове сканування та перезаписує кеш.

## Сортування та підтримувані файли

Манґи, глави та сторінки сортуються природно: числові частини назви
порівнюються як числа. Наприклад:

```text
1.jpg
2.jpg
10.jpg
```

Перевірка розширень зображень наразі реалізована в `ChaptersActivity`.
Додаючи новий формат, потрібно також перевірити, що його підтримує Glide.

## Перевірка змін

Мінімальний ручний сценарій перевірки:

1. Зібрати й запустити застосунок на API 24-29 та API 30+.
2. Вибрати бібліотеку з кількома манґами.
3. Перезапустити застосунок і перевірити, що список манґи відновився без
   повторного вибору директорії та сканування.
4. Повторно вибрати кореневу директорію та перевірити оновлення кешу.
5. Перевірити природне сортування назв із числами.
6. Відкрити манґу з директоріями глав.
7. Відкрити манґу без директорій глав.
8. Перевірити свайпи, масштабування, подвійний дотик і приховування панелі.
9. Перевірити зміну орієнтації.
10. Перевірити порожні директорії та директорії з непідтримуваними файлами.

У репозиторії поки немає активних unit- чи instrumented-тестів. Для подальшого
розвитку варто насамперед покрити тестами природне сортування, визначення
зображень і перетворення структури директорій у моделі `MangaItem` та `Chapter`.

## Відомі технічні особливості

- Activities самостійно керують `CoroutineScope`; scope скасовується в
  `onDestroy`.
- Моделі `MangaItem` і `Chapter` оголошені поруч із відповідними Activities.
- Природне сортування дублюється в `MainActivity` і `ChaptersActivity`.
- Список URI сторінок передається в `ReaderActivity` через `Intent`; для дуже
  великих глав це може наблизитися до ліміту Binder transaction.
- Для Android 11+ застосунок запитує `MANAGE_EXTERNAL_STORAGE`, хоча вибір
  директорії через SAF зазвичай може працювати без повного доступу до всіх
  файлів. Перед публікацією в Google Play політику дозволів слід переглянути.
- Частина коментарів у Kotlin/XML має пошкоджене кодування; це не впливає на
  виконання, але ускладнює підтримку.
